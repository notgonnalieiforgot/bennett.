# Projected-MRR Handoff Docs

Every project ships with a projected-MRR financial sheet, delivered as a native Google Doc under `pat@sidechained.net` with full editor access, as a standing part of handoff.

## Why it matters
- The user set this as a standing expectation: each site is treated as a revenue engine, and the financial picture is part of every handoff — not an afterthought.
- It makes the value of the build legible in dollars, which anchors the [[Growth Partner Model]] and the scaling conversation that follows.
- A grounded projection (traffic → leads → close rate → job value → recurring/repeat, with conservative/base/upside scenarios) keeps expectations honest and shows rigor.

## How we pursue it
- [[Project Handoff Playbook]] — the MRR doc is step 3 of the standing close-out, triggered automatically at the end of a build.
- [[Growth Partner Model]] — the projection is the artifact that turns "here's your site" into "here's how we grow your revenue."
- Ground the model in the site's actual funnel; produce conservative / base / upside scenarios rather than a single number.
- Delivery mechanics: upload with `contentMimeType: text/plain` so Drive auto-converts to a native, editable Doc (text/html does NOT convert). Since pat@sidechained.net owns the Drive, the file already carries full editor access.
