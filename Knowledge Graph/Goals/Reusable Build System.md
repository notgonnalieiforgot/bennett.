# Reusable Build System

Turn every build into reusable skills and playbooks so each new site starts from a system, not from scratch — capturing motion stacks, design tokens, integrations, and process once and reapplying them everywhere.

## Why it matters
- Starting from a system is faster, more consistent, and higher quality than rebuilding from zero each time.
- It makes the [[Growth Partner Model]] viable at scale: you can take on more companies without linear effort.
- Lessons from each project (gotchas, patterns, components) become assets instead of evaporating after handoff.
- It feeds and is fed by the [[Visual Knowledge Graph]] — the system lives as connected, queryable notes.

## How we pursue it
- [[Sidechained Skills Library]] — the private skill repo where reusable Agent Skills are aggregated and installed into Claude Code.
- [[Agency Build Playbook]] — the canonical start-to-finish process (audit → brand/tokens → scaffold → Framer → Higgsfield → motion → deploy) compiled from the flagship.
- [[High-End Interactive Website Builder]] — the premium, scroll-driven agent process for award-caliber animated sites, with [[UI-UX Pro Max]] as its design-intelligence layer.
- [[Landing-Page Motion Framework]] — the reusable scroll/motion stack (hooks + components + skills) reused instead of rebuilt for new animated pages.
- [[Construction Trade Website Lifecycle]] — a templated end-to-end workflow for contractor marketing sites + ops dashboards.
- [[Superpowers Plugin]] — reusable skill infrastructure that extends what the build system can do.
