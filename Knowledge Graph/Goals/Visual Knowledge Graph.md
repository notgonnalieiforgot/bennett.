# Visual Knowledge Graph

Grow a richly connected Obsidian graph of every process, skill, workflow, goal, and project (the BENNETTMAIN initiative) so knowledge compounds, stays queryable, and the relationships between the work become visible.

## Why it matters
- Scattered knowledge doesn't compound; a linked graph turns one-off lessons into a reusable, navigable system.
- Graph view makes structure visible — clusters reveal which goals, workflows, and projects reinforce each other.
- A queryable second brain feeds the [[Reusable Build System]] and the [[Growth Partner Model]]: faster, more consistent delivery across clients.
- The denser the cross-links, the more useful the graph; liberal `[[Wikilinks]]` are the point, not decoration.

## How we pursue it
- [[graphify]] — the CLI that turns any input (code, docs, papers, media) into a knowledge graph, the engine behind this vault.
- [[Knowledge Graph Workflow]] — the repeatable process for capturing notes and wiring them into the existing graph.
- [[Knowledge Graph Home]] — the entry point / top-level map that orients navigation across the vault.
- [[Reusable Build System]] — the graph is where reusable skills and playbooks get indexed and connected.
- Every note links liberally to MOCs and siblings so no node is an island.
