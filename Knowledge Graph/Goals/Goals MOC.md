# Goals MOC

The hub for Sidechained's overarching goals — the outcomes every project and process is meant to advance. These goals reinforce each other: a reusable build system enables the growth-partner model, the knowledge graph makes both compound, and financial handoffs make the value legible.

## Goals
- [[Growth Partner Model]] — be a partner who scales companies after delivery, not a one-off builder.
- [[Projected-MRR Handoff Docs]] — ship a projected-MRR Google Doc with editor access as part of every handoff.
- [[Visual Knowledge Graph]] — grow a connected Obsidian graph so knowledge compounds and stays queryable.
- [[Reusable Build System]] — turn every build into reusable skills + playbooks so new sites start from a system.

## How the goals connect
- [[Reusable Build System]] makes the [[Growth Partner Model]] scalable; the [[Visual Knowledge Graph]] makes both compound; [[Projected-MRR Handoff Docs]] makes the value of the work legible in revenue terms.
