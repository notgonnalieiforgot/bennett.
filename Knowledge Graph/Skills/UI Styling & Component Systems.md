# UI Styling & Component Systems

The component-and-tokens layer of a build: how interfaces get assembled from accessible primitives and a consistent token system rather than one-off CSS. Reach for these once direction is set and it's time to ship real, maintainable UI.

## What it does
- **ui-styling** — accessible UIs with shadcn/ui (Radix + Tailwind), utility-first Tailwind, dark mode, responsive layouts, and canvas-based visuals.
- **shadcn** — shadcn/ui CLI, component installation, composition patterns, custom registries, and theming.
- **design-system** — design tokens and reusable component-library architecture so patterns stay consistent across pages and projects.

## Used in / pairs with
- [[shadcn ui]] — the Radix + Tailwind component layer these skills install and compose.
- [[Tailwind v4]] — the CSS-first token system tokens are expressed in.
- [[UI-UX Pro Max]] — supplies the styles, palettes, and font pairings this layer applies.
- [[Frontend Design & Taste Skills]] — keeps the assembled components from drifting into generic defaults.
