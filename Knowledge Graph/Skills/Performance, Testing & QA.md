# Performance, Testing & QA

The cluster of skills that close the gap between "code that runs" and "code that ships confidently" — covering bundle optimization, end-to-end verification, component quality, diff review, and security auditing. Reach for it at the delivery gate of any phase or before any handoff where Core Web Vitals or correctness matter (LCP < 2s, CLS 0, INP < 100ms).

## What it does
- **web-performance-optimization** — code splitting, lazy loading, caching, service workers, bundle reduction, Core Web Vitals.
- **webapp-testing** — Playwright-driven local web app testing, screenshots, browser logs.
- **verification** — full-story end-to-end verification: browser to API to data to response.
- **verify** — run the app and observe behavior to confirm a change or fix actually works.
- **run** — launch and drive the project's app to see a change working in the real environment.
- **react-best-practices** — TSX component quality checklist covering structure, hooks, a11y, performance, and TypeScript.
- **code-review** — review the current diff for correctness bugs at a chosen effort level.
- **security-review** — security-focused review of changes for vulnerabilities and exposure.
- **review** — review a pull request for overall correctness and quality.
