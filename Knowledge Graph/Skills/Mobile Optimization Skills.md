# Mobile Optimization Skills

Two complementary skills — mobile-optimize and mobile-audit-fix — for making any web project feel like a native, bug-free mobile app. Reach for mobile-optimize to build mobile-first UX from scratch, and mobile-audit-fix to diagnose and repair existing mobile breakage.

## What it does
- **mobile-optimize** — builds world-class, native-feeling mobile UX: thumb-zone layout, touch interactions, swipe gestures, bottom nav, pull-to-refresh, native-like transitions, and mobile performance, either from scratch (mobile-first architecture) or by converting a desktop site.
- **mobile-audit-fix** — systematically audits, fixes, and verifies mobile compatibility issues: broken animations on phones, iOS Safari quirks, Android rendering, viewport/safe-area-inset, touch targets, `-webkit-` vendor prefixes, and IntersectionObserver problems.
- Covers all frontend stacks — plain HTML/CSS/JS, React/JSX, Vue, Svelte, Astro, Next.js, and static site generators.

## Used in / pairs with
- [[Landing-Page Motion Framework]] — premium scroll motion must degrade gracefully on mobile; these skills harden the reused stack for phones (and pair with `use-prefers-reduced-motion`).
- [[Next.js 16]] — the primary framework these skills tune and audit for mobile.
