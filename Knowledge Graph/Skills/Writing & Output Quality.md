# Writing & Output Quality

Two orthogonal output-discipline skills: one scrubs AI writing tells from prose so copy reads as genuinely human, the other enforces complete unabridged code and content generation so nothing is silently truncated or stubbed. Use them together on any client-facing written deliverable.

## What it does
- **humanizer** — strips AI-writing tells (em-dash overuse, rule-of-three, filler phrases, vague attributions) for natural human prose that passes scrutiny in client copy and marketing content.
- **output-skill** — overrides LLM truncation behavior; enforces complete, unabridged code and output with no placeholder patterns, ellipsis stubs, or "rest of code here" shortcuts.

## Used in / pairs with
- [[Communication Tone]] — humanizer aligns with the studio's casual, goal-driven dialect so AI-assisted copy matches the defined voice.
- [[Brand & Identity Skills]] — brand voice guidelines depend on copy that actually sounds like the brand, not a language model.
- [[Frontend Design & Taste Skills]] — output-skill ensures generated component code is always complete and production-ready, never stubbed.
