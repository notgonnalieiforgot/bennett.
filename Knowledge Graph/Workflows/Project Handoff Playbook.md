# Project Handoff Playbook

The standing play run proactively at the end of every project (without waiting to be asked): deploy with a live link, deliver a full final audit, and create a projected-MRR Google Doc for the client. Each site is treated as a revenue engine, so the handoff frames the growth picture for scaling the company.

## Steps / shape
1. **Deploy to production** and hand over the live link; **redeploy** whenever changes land so the link always reflects latest.
2. **Full final audit** — design, motion + scroll animations, SEO, the admin dashboard (overview plus a real audit), security, performance, and accessibility. Evidence-based (build output, tests, real checks); call out gaps honestly.
3. **Projected-MRR financial sheet** — see [[Projected-MRR Handoff Docs]]. Upload into a Google Doc under `pat@sidechained.net` with full editor access. Ground the MRR projection in the funnel (traffic → leads → close rate → job value → recurring/repeat) across conservative/base/upside scenarios.

> Doc gotcha: upload with `contentMimeType: text/plain` so Drive auto-converts to a native, editable Doc (`text/html` does not convert). Get the mime type right on first create — there is no rename/delete tool. Link form: `https://docs.google.com/document/d/<id>/edit`.

## Applied in
- Every project — this is the universal closing stage of the [[Agency Build Playbook]].
